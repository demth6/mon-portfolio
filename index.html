Definitions des elements du schema
pfSense
Pare-feu / Routeur central
Solution open source basee sur FreeBSD faisant office de coeur du reseau. Il filtre tout le trafic entrant et sortant, gere le routage entre les zones (LAN, DMZ, VPN), effectue le NAT et heberge le serveur VPN OpenVPN. Trois interfaces : RE0 (WAN), RE1 (LAN), RE2 (DMZ).
IP LAN : 10.1.1.1/29 · igc0/igc1/igc2
Switch
Commutateur reseau manageable
Equipement reseau NETGEAR qui distribue le trafic entre les differents equipements du LAN. En tant que switch manageable, il prend en charge les VLANs : chaque port peut etre assigne a un VLAN different, permettant de segmenter logiquement le reseau physique en plusieurs zones isolees.
IP : 10.1.1.2/29 · VLAN 21 + VLAN 31
DMZ
Zone demilitarisee
Zone reseau isolee entre Internet et le LAN interne. Elle heberge les serveurs accessibles depuis l'exterieur (serveurs web). Si un attaquant compromet un serveur en DMZ, il ne peut pas acceder directement au LAN interne grace aux regles de pfSense. C'est une mesure fondamentale de defense en profondeur.
Reseau DMZ : 192.168.51.0/29 (6 IP utiles)
VLAN
Reseau local virtuel
Segmentation logique du reseau physique. Le VLAN 21 isole les serveurs Proxmox, le VLAN 31 isole le Wi-Fi. Sans VLAN, un appareil Wi-Fi compromis pourrait acceder directement aux serveurs. Avec les VLANs, pfSense controle et filtre le trafic inter-zones.
VLAN 21 : 192.168.21.0/24 · VLAN 31 : 192.168.31.0/24
Proxmox VE
Hyperviseur de type 1
Hyperviseur open source base sur KVM (Kernel-based Virtual Machine). Il permet de faire tourner plusieurs machines virtuelles sur un seul serveur physique. Interface web centralisee pour creer, gerer et superviser les VMs. Type 1 signifie qu'il tourne directement sur le materiel, sans OS intermediaire.
VLAN 21 · 7 VMs deployees (AD, GLPI, Zabbix, Artica, XiVO, Unifi)
Hyper-V
Hyperviseur Microsoft (DMZ)
Hyperviseur de type 1 integre a Windows Server, utilise cote DMZ. Il heberge les serveurs web SRV-WEB1 et SRV-WEB2 exposes vers Internet, ainsi que SRV-ZEVENET. Choix complementaire a Proxmox permettant de travailler sur les deux technologies de virtualisation majeures du marche.
IP Hyper-V : 192.168.51.2/29
VPN Tunnel
OpenVPN — Acces distant securise
Tunnel chiffre configure sur pfSense via OpenVPN, permettant a un utilisateur distant (PC Distant) d'acceder au reseau interne de facon securisee via Internet. Une autorite de certification interne (CA-PRIME) a ete creee pour authentifier les connexions par certificats plutot que par simple mot de passe.
Tunnel VPN : 172.2.1.0/24 · Protocole UDP
Borne Wi-Fi
UniFi U7 — VLAN 31
Point d'acces sans fil UniFi U7 d'Ubiquiti, connecte sur les ports dedies au VLAN 31 du switch. Les appareils Wi-Fi sont isoles dans le VLAN 31 (192.168.31.0/24) et ne peuvent pas acceder directement aux serveurs du VLAN 21. Gere par le controleur logiciel SRV-UNIFI (VM Proxmox).
IP : 192.168.31.1/24 · VLAN 31 dedie
Active Directory
Annuaire Microsoft — SRV-AD1 + AD2
Service d'annuaire Microsoft centralisant la gestion des utilisateurs, ordinateurs et ressources. SRV-AD1 (192.168.21.1) est le controleur de domaine principal avec Exchange pour la messagerie. SRV-AD2 (192.168.21.2) est le controleur secondaire assurant la redondance : si AD1 tombe, AD2 prend le relai automatiquement.
SRV-AD1/Exchange : 192.168.21.1/24 · SRV-AD2 : 192.168.21.2/24
GLPI
Gestion de parc & Helpdesk
Logiciel open source de gestion du patrimoine informatique et des tickets de support (ITSM). Deploye sur Ubuntu Server 24 LTS via script Bash automatise. Gere l'inventaire de tous les equipements (ordinateurs, imprimantes, licences) et le cycle de vie des incidents signales par les utilisateurs.
SRV-GLPI : 192.168.21.3/24 · Pile LAMP
Zabbix
Supervision reseau temps reel
Outil open source de monitoring qui surveille en temps reel l'etat de toute l'infrastructure : CPU, RAM, disque, disponibilite des services. Des agents Zabbix installes sur chaque VM remontent les metriques. Les triggers declenchent des alertes automatiques si un seuil est depasse (ex: CPU > 90%, serveur inaccessible).
SRV-ZABBIX : 192.168.21.4/24 · SNMP + Agents
Artica Proxy
Filtrage web applicatif
Proxy web open source qui filtre les acces Internet des utilisateurs au niveau applicatif HTTP/HTTPS. Bloque les sites dangereux ou non autorises, journalise la navigation, optimise la bande passante via le cache. Complement de pfSense : pfSense filtre au niveau reseau (couche 3), Artica au niveau application (couche 7).
SRV-ARTICA : 192.168.21.5/24 · Proxy transparent
XiVO
Telephonie sur IP (VoIP)
IPBX open source gerant la telephonie d'entreprise via le protocole SIP (Session Initiation Protocol) sur le reseau IP existant. Remplace le cablage telephonique classique. Gere les extensions, les files d'attente, les transferts. Le protocole RTP transporte la voix. Les postes SIP ou softphones s'enregistrent aupres de XiVO.
SRV-XIVO : 192.168.21.6/24 · SIP port 5060 · RTP
Zevenet
Load Balancer — Haute disponibilite
Repartiteur de charge open source en DMZ. Distribue les connexions entrantes entre SRV-WEB1 et SRV-WEB2 selon un algorithme round-robin. Des health checks reguliers verifient la disponibilite de chaque serveur. Si l'un tombe, Zevenet bascule automatiquement tout le trafic vers l'autre : les utilisateurs ne voient aucune interruption.
SRV-ZEVENET : 192.168.51.5/29 · Algo round-robin
PC Administrateur
Poste d'administration dedie
Poste dedie a l'administration de toute l'infrastructure, sur le reseau physique 10.1.1.0/29. Depuis ce poste, l'administrateur accede a l'interface pfSense, a Proxmox, aux VMs via SSH ou RDP, et a tous les outils de gestion. Les regles pfSense limitent cet acces privilegié a cette seule machine.
IP : 10.1.3/29 · Acces privilege
SRV-UNIFI
Controleur Wi-Fi Ubiquiti
Machine virtuelle hebergeant le logiciel controleur UniFi d'Ubiquiti. C'est depuis cette interface centralisee que sont configures les SSIDs (noms de reseaux Wi-Fi), les mots de passe, les VLANs associes et les politiques d'acces sans fil. Permet de gerer plusieurs bornes depuis un seul endroit. Distinct de la borne physique U7.
SRV-UNIFI : 192.168.21.7/24
Zones du schema et leur role
Zone Wi-Fi (VLAN 31)
Reseau sans fil isole. La borne UniFi U7 (192.168.31.1/24) diffuse le signal Wi-Fi. Les appareils connectes (smartphones, tablettes) sont segmentes du LAN serveurs pour la securite.
Zone Serveurs (VLAN 21)
Infrastructure Proxmox avec 7 VMs : AD1/Exchange, AD2, GLPI, Zabbix, Artica, XiVO, Unifi. Reseau 192.168.21.0/24. Zone interne protegee par pfSense et le filtrage VLAN.
Zone DMZ — Serveurs web
Zone exposee a Internet via pfSense (RE2). Heberge Hyper-V avec SRV-WEB1, SRV-WEB2 et SRV-ZEVENET en 192.168.51.0/29. Isolee du LAN par des regles de filtrage strictes.
Acces distant VPN
Le PC Distant se connecte via tunnel OpenVPN sur pfSense (RE0/WAN). Le tunnel 172.2.1.0/24 chiffre les communications et donne acces au reseau interne comme si l'utilisateur etait sur site.
Machines virtuelles Proxmox — VLAN 21 (192.168.21.0/24)

Veille technologique
Windows 11 en entreprise
Migration, securite, nouveautes — surveillance des evolutions Microsoft
Pourquoi cette veille

Directement liee a mon stage chez FixProTech ou j'ai realise des migrations Windows 10 vers Windows 11 sur materiel non compatible. Comprendre les exigences et evolutions de Windows 11 m'etait indispensable pour mener ces interventions de facon efficace et securisee.

Points cles surveilles
🔒
TPM 2.0 & Secure Boot
Puce de securite obligatoire. Stocke les cles BitLocker, garantit l'integrite du demarrage. Absente sur machines avant 2018.
🛡️
Credential Guard
Isole les identifiants Windows (NTLM, Kerberos) dans un environnement virtualise securise VBS.
🤖
Copilot IA (24H2)
Integration native de l'IA dans Windows 11 24H2. Assistant contextuel, actions systeme automatisees.
⚠️
Fin support Win10
Support officiel Windows 10 termine en octobre 2025. Risque de failles non corrigees pour les entreprises.
🔑
Suppression NTLMv1 (25H2)
Windows 11 25H2 supprime NTLMv1, vulnerable aux attaques pass-the-hash. Renforcement securite reseau.
🛠️
Migration Flyby11
Outil open source contournant les verifications TPM/CPU via l'installeur Windows Server. ISO officielle + donnees conservees.
Lien avec mon stage FixProTech
💼
Ma veille m'a permis de comprendre pourquoi les machines echouaient au controle Windows 11. J'ai utilise Flyby11 avec l'ISO officielle pour contourner les verifications TPM 2.0 et CPU, tout en conservant les donnees clients. Ma veille sur la fin de support Windows 10 m'a aide a argumenter l'urgence de migrer aupres des clients.

Veille juridique
Cadre legal du numerique
RGPD · NIS 2 · LCEN — obligations de l'administrateur systemes
Pourquoi cette veille

En tant que futur administrateur systemes et reseaux, je suis directement concerne par le cadre legal. Je configure les pare-feux, gere les logs de connexion, administre les comptes — toutes ces actions ont des implications juridiques que je dois maitriser.

Les 3 textes fondamentaux
RGPD
En vigueur depuis 2018
Reglement General sur la Protection des Donnees
Obligation de proteger les donnees par chiffrement et controle des acces
Droit des utilisateurs : acces, rectification, suppression
Notification CNIL obligatoire en cas de fuite sous 72h
Sanctions : jusqu'a 4% du CA mondial ou 20 millions d'euros
Impact admin : Les logs reseau contiennent des donnees personnelles — conservation encadree, acces restreint
NIS 2
Transposition oct. 2024
Directive Network and Information Security 2
Perimetre elargi a de nombreux secteurs critiques
Notification d'incident significatif a l'ANSSI sous 24 heures
Audits de securite reguliers et plans de continuite obligatoires
Mesures techniques : chiffrement, MFA, gestion des acces
Impact admin : pfSense et Zabbix doivent detecter et documenter les incidents rapidement
LCEN
En vigueur depuis 2004
Loi pour la Confiance dans l'Economie Numerique
Conservation obligatoire des logs de connexion pendant 1 an
Transmissibles aux autorites judiciaires sur requisition
Interdiction formelle de lire les mails personnels des salaries
Responsabilite des hebergeurs sur les contenus illicites
Impact admin : pfSense et Artica journalisent toutes les connexions — logs conserves 1 an minimum
